# Story 1.1: App Shell & Navigation

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a user,
I want to open the One Down app and see a clean interface with navigation controls,
so that I have a usable app foundation.

## Acceptance Criteria

1. **Given** a fresh install of the app, **when** the user launches the app, **then** they see a top bar with a task list icon (top-left), a star box placeholder (top-second-to-left), and a settings icon (top-right).
2. **Given** the app shell is rendered, **when** the user views the home screen, **then** a floating add button is visible in the bottom-right corner.
3. **Given** the app is running on Android, **when** the device orientation changes, **then** the app remains locked to portrait orientation (no landscape rendering).
4. **Given** the app shell is rendered, **when** it is laid out on devices with notches, status bars, or home indicators, **then** the top bar, floating add button, and home content respect the safe area insets and never overlap system UI.
5. **Given** the project uses NativeWind, **when** any shell component is styled, **then** styling is applied via NativeWind/Tailwind utility classes (no inline `StyleSheet` styles for new shell components — the temporary scaffold styles introduced in Story 1.0 may be removed as part of this work).
6. **Given** the app shell uses gluestack-ui v3 components, **when** `bun --cwd apps/mobile start` is run, **then** the app renders without runtime errors and `bun run typecheck`, `bun run lint`, `bun run format:check`, and `bun run test` all pass.
7. **Given** Reanimated 4 and `react-native-gesture-handler` are configured, **when** Metro bundles the mobile app, **then** the Babel config loads `react-native-worklets/plugin` last and the gesture handler root view wraps the app, satisfying the prerequisites for Story 1.3's swipeable card stack.
8. **Given** the shell has interactive icons (task list, star box, settings, floating add), **when** a screen reader (TalkBack) focuses on each, **then** each control announces a descriptive accessibility label and exposes the correct accessibility role (button) with a minimum 44x44pt touch target.
9. **Given** the user has not yet added tasks, **when** they tap the task list icon, the star box, the settings icon, or the floating add button, **then** the tap target acknowledges the press (visual press state) — no destination screens are required for this story; logging a no-op action or routing to a placeholder route is acceptable so long as the app does not crash.

## Dev Notes

This story is the first user-visible UI story. It establishes the persistent app shell (top bar + floating add button + safe areas) and the styling toolchain (NativeWind, gluestack-ui v3) that every later story in Epic 1 builds on. **It must not implement any task logic, navigation destinations, or stack/card behaviour** — Stories 1.2 through 1.5 own those.

### Scope Boundaries

- **In scope:** Persistent shell layout, NativeWind/Tailwind setup, gluestack-ui v3 init + minimal primitives, Reanimated/gesture-handler tree-level prerequisites, mobile test framework setup, accessibility labels, safe area + portrait verification.
- **Out of scope:** Quick-add input/forms (Story 1.2), task data model, expo-sqlite/Drizzle wiring (Story 1.2), card stack/swiping (Story 1.3), inline editing (Story 1.4), task overview list (Story 1.5), navigation destinations behind the icons (the icons are tappable but do not need to navigate anywhere yet — see AC 9), Zustand stores, tRPC, AI, sync, auth, PostHog instrumentation.
- **Do not** create `task/`, `brain-dump.tsx`, `triage.tsx`, `settings.tsx`, or `(auth)/` route files yet — they belong to later stories. Empty placeholder routes are unnecessary; tap targets in this story can route to a no-op handler or `router.push("/")`.
- **Do not** add Drizzle, expo-sqlite, Zustand, TanStack Query, tRPC, Supabase, PostHog, or RevenueCat in this story.

### Accessibility Requirements

Per the UX spec (`ux-design-specification.md#Accessibility-Strategy`), every shell control MUST:

- Provide a descriptive `accessibilityLabel` (e.g. "Open task list", "View star activity", "Open settings", "Add task"). Avoid visual labels like "list button".
- Provide `accessibilityRole="button"` on the icon controls and FAB.
- Meet 44x44pt minimum touch target — gluestack `Pressable` defaults are insufficient unless paired with explicit padding utilities; ensure each tap area uses `min-h-[44]` and `min-w-[44]` (or larger).
- Star box placeholder uses `accessibilityLiveRegion="polite"` so when Epic 4 wires up actual values, screen readers announce changes without interrupting.
- Skip `console.log` for press feedback — the lint rule is `no-console`.

### Reanimated 4 / Gesture Handler Pre-wiring

Story 1.3 owns the swipeable card stack but depends on this story laying the groundwork:

1. `react-native-worklets/plugin` is the LAST entry in `babel.config.js` plugins. (UX spec + Reanimated 4 docs.)
2. The root layout wraps children in `<GestureHandlerRootView style={{ flex: 1 }}>`. Without this, gestures will silently fail.
3. Do NOT add Reanimated worklets, animations, or gestures in this story. The shell controls use plain `Pressable` press states.

If the Reanimated mock is missing in tests, jest will fail with "Reanimated 2/3/4 failed to create a worklet". Add `jest.mock('react-native-reanimated', () => require('react-native-reanimated/mock'))` (or follow the Reanimated 4 testing guide).

### Risks / Things Future Stories Should Be Aware Of

1. **Manual gluestack-style primitives.** `src/components/ui/*.tsx` are hand-rolled to satisfy the gluestack-ui v3 copy-paste boundary. If a future story needs a richer gluestack component (e.g. `Toast`, `Modal`), it should run `npx gluestack-ui init` then `add <component>` and reconcile the providers — the `@/components/ui/...` import paths will still match the gluestack CLI output.
2. **Single Babel plugin makes "LAST" trivially true.** When a future story adds another Babel plugin, it must keep `react-native-worklets/plugin` last in the array. Reanimated 4 silently fails without it.
3. **No `GluestackUIProvider` is mounted yet.** Manual primitives don't need it. The story that introduces a real gluestack component must add the provider in `_layout.tsx`.
4. **Star box is a placeholder.** Epic 4 (Story 4.2) replaces its body content. Its outer `Pressable` + `accessibilityLiveRegion="polite"` already supports announce-on-update, so the contract is stable.
